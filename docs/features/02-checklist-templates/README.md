# Checklist Templates

## Purpose

Checklist templates provide reusable release assurance starting points for common project types.

## Business Value

- reduces repeated checklist design work,
- improves consistency across teams,
- accelerates onboarding into the release process,
- makes expected verification more visible earlier in delivery.

## Users Involved

- release owners,
- developers and technical owners,
- security, platform, and operations reviewers.

## Core Workflow

1. The organization defines reusable templates.
2. A project starts from the most relevant template.
3. Gatekeeper combines the template with project profile data.
4. The resulting checklist becomes the working release assurance set for that project.

## Key Business Rules

Templates should support common project types such as:

- public web application,
- public API,
- internal tool,
- background service,
- high-risk or regulated project.

Templates should create a consistent baseline, but they should still work with applicability rules rather than acting as rigid one-size-fits-all checklists.

## Outputs

- initial checklist structure,
- consistent baseline controls by project type,
- faster project setup.

## Dependencies

- project profiles,
- control library,
- applicability rules.

## Out of Scope

Templates do not remove the need for project-specific review where risk, architecture, or regulation requires additional controls.
